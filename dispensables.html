Duplicate Code

Code smell ini terjadi apabila terdapat dua atau lebih method atau class yang sangat identik. Solusi: Gunakan extract method untuk tempat ditemukannya duplicate code.

Before

After

Pada contoh di atas, kita telah memperbaiki kode dengan menghilangkan variabel yang tidak perlu (luas) dan langsung return hasil perhitungan.

Lazy Class

Code smell ini terjadi apabila terdapat sebuah class dengan beberapa method yang hanya melakukan sedikit proses atau bahkan tidak melakukan proses apapun Solusi: Gunakan inline class untuk memindahkan semua fitur dari lazy class ke class lain.

Before

After

Pada kelas address diatas, tidak terdapat banyak fungsionalitas, hanya sekedar untuk menyimpan alamat saja Untuk mengatasi masalah ini, dapat melakukan refaktorisasi dengan metode inline class ke class customer.

Data Class

Code smell ini terjadi apabila terdapat sebuah class yang hanya memiliki sedikit fungsionalitas dan tidak dapat dikatakan sebagai sebuah class. Solusi: Gunakan Encapsulation Field untuk variable public. Untuk data array, menggunakan Encapsulate Collection.

Before

After

Pada kelas diatas, dapat dilihat bahwa kelas tersebut hanya menyimpan atribut beserta dengan setter getter nya. Hal ini diklasifikasikan ke dalam data class dan untuk merefaktorisasi kelas tersebut dapat menambahkan 1 atau 2 method baru.

Dead Code

Code smell ini terjadi apabila terdapat sebuah class atau method yang sama sekali tidak digunakan dalam program tersebut. Solusi: dapat menggunakan inline class atau hierarchy collapse serta dapat juga menghapus code yang tidak digunakan.

Before

After

Pada class calculator diatas, terdapat sejumlah function untuk melakukan operasi matematika. Namun untuk funct ion perkalian dan pembagian tidak pernah digunakan sama sekali. Jadi kita dapat menghapus kedua function tersebut dengan remove method.

Speculative Generality

Code smell ini terjadi apabila terdapat penambahan method ke dalam sebuah class yang mungkin akan digunakan di masa depan. Hal ini menyebabkan kode menjadi semakin kompleks dan sulit untuk dikelola. Solusi: Menggunakan collapse hierarchy, inline class, inline method, remove parameter, dan hapus field yang tidak terpakai.

Before

After

Pada contoh setelah perbaikan, kita hanya membuat kelas-kelas yang benar-benar dibutuhkan tanpa adanya interface Shape. Ini membuat kode menjadi lebih sederhana, lebih mudah dipahami, dan menghindari smell code "Speculative Generality".
